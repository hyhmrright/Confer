# PRD — 想法 A｜个人对外委托助理（审批 inbox）

> 文档状态：MVP 验证版（Wizard-of-Oz 探针，**非自主 agent**）｜阶段：Wave-2 锚点
> 全局赌注（leap-of-faith）：「人们愿意让自己的 Agent 去和别人的 Agent 来回沟通把事办成，而不是自己亲自做。」
> **非创始人闸门（硬规则，贯穿全文每个验证环节）**：创始人用自己两个账号自测**只证明管道通，不证明赌注成立**。成功**只**在 **≥2 名独立、非创始人的真实人类**完成**非脚本化、有真实后果**的来回时才计数。**所有创始人双账号交互必须在所有成功计数中被显式审计剔除**——这条规则在第 3、5、8、9 节各自重申，任何指标的分母都先过这道审计。

---

## 1. 一句话定位 + portfolio 角色

**定位**：你的「对外委托助理」——你把一件要联系外部人/商家的杂事交给它，它在幕后跑腿，只在需要你判断时推一张决策卡（**通过 / 改价 / 拒绝**），办完交付结果。

**portfolio 角色**：A 是 **Wave-2 的杀手级应用锚点**，把抽象的 A2A 协议价值翻译成普通人一眼能懂的「替我办事」。它不依赖双边 agent 网络（冷启动单边消费），因此被当作「委托价值是否真实存在」的最干净 desirability 验证场。

**前置依赖（硬）**：A 是 Wave-2，**只有 Wave-1（C/B）通过「全局赌注可解释性闸门」后才启动**——即 C 已有 **≥2 对独立非创始人**完成有后果的来回（**审计剔除创始人账号**），且 ≥40% C/B 受访者浮现真实跨主人对手方需求。**未过此闸门，A 不按本设计启动。**

## 2. 问题 / JTBD

### JTBD

> 当我有一件**必须联系外部人/商家、且会有几个来回**的杂事（打电话改约、邮件催办、比价后预订、争议一笔扣款）时，我想**把跑腿和盯流程的活儿甩出去**，这样我**只在真正需要拍板的那一刻花 10 秒做个决定**，不用自己从头盯到尾。

### 基于 A value 假设的核心承诺（A-v1）

委托的「难」不在于把事办成本身，而在于**来回的协商与判断**。如果助理替你跑完所有腿、**只在需判断时推一张决策卡**，就交付了人们想从「委托」里得到的大部分价值。

### 诚实写出反方 idea-killer（第 1 号 leap-of-faith，必须正面验证）

| 反方论点 | 为什么致命 |
|---|---|
| **便宜杂事 → 决策卡是负价值** | 订位、比价这类杂事，成本是「开始的那 30 秒」，不是协商。若助理为一件你一通电话就能办完的事推了 4 张卡，**感知价值是负的**——从「解脱」变成「唠叨」。 |
| **贵杂事 → 用户不放权** | 真正值得委托、协商成本高的杂事（争议扣款、谈租金、复杂退货），恰是用户**因利害与上下文太重而拒绝交出判断权**的。 |
| **合起来的死局** | 助理只抓住「便宜中间地带」（边际价值薄，且 OpenTable / 比价网已办了），却在「有价值的两个极端」被解雇。价值与可靠性指向相反（A-g2）。 |

**本 PRD 的第一性目标，就是用最便宜的方式判定上面这个死局是否成立。在它被 A-EXP2 证伪之前，不写任何自主 agent 代码。**

## 3. 目标用户 + 反向用户

### 目标用户（beachhead）

- **已经在外包杂事的忙碌专业人士 / 小生意主**：每周有一定量「打电话、发邮件、比价、预订」的对外杂事，对「替我办事」有现成心智。
- 筛选器（A-EXP1）：3 题问卷，问电话/邮件类对外杂事频率。
- 关键特征：杂事**有几个来回**（需要判断），而非一键即完成。

### 反向用户（明确不服务，且用于保护验证纯度）

| 反向用户 | 为什么排除 |
|---|---|
| **创始人本人 / 团队 / 任何创始人控制的双账号** | 自测只证明管道通。**所有创始人交互从每一项成功计数审计剔除**（非创始人闸门，全文硬规则）。招募名册须独立于创始人社交图，每个计入成功的用户须可核验为独立非创始人。 |
| **只做「一键即完成」杂事的人** | 无来回 = 无判断 = 无决策卡价值（A-v1 反方的「便宜中间地带」）。 |
| **高利害、强上下文、绝不放权的人**（打官司、谈大额合同） | 结构性拒绝委托判断（A-v1 反方的「贵极端」）；MVP 不试图说服他们，而是**记录他们拒绝的边界在哪**作为发现数据（US-5）。 |

## 4. 要 de-risk 的 leap-of-faith

直接引用 A 的 `leapOfFaith`，按 `rankedTestOrder` 前四：

| 排序 | ID | 类别 | 假设（精简） | impact / uncertainty |
|---|---|---|---|---|
| 1 | **A-v1** | value | 难点在来回协商/判断；只在需判断时推决策卡（通过/改价/拒绝）即交付委托的大部分价值。**核心承诺 + 阶段闸门。** | high / high |
| 2 | **A-v2** | value | 对外杂事复发频率足够高，值得设立并保留一个「常驻授权助理」，而非每次临时找通用助理。 | high / medium |
| 3 | **A-f3** | feasibility | **授权边界可靠性**：自主 agent 替用户办事时能可靠待在授权边界内——不越界、不被敌意网站/对手 prompt-injection。 | high / high |
| 4 | **A-vi1** | viability | 消费者愿付价格能覆盖每次 errand 的可变成本（LLM token + 动作执行 + 可能的人工兜底）并为正毛利。 | high / high |

### ⚠️ A-f3 是灾难性、不可逆风险（与 A-f1 区分）

- **A-f1（对人/对网站执行动作）不是 leap-of-faith**：仓库已证明它纯属**未建**，是已知的 greenfield 范围事实（低不确定，无须「发现」）。MVP 不自建它（见 OUT-1）。
- **A-f3 才是真不确定**：`classifyPermissionLevel`（`engine.ts`）是**纯字符串前缀匹配**（`payment` / `sign_contract` → L3，read → L1，其余 → L2），边界可被「动作如何措辞」绕过；内容审核只守 A2A peer 图，**不守任意 web/email/phone 表面**。一旦 agent 读取不可信内容并能执行动作，prompt-injection 是活的、未解决的威胁。
- **后果**：一次越界动作若花了钱或替用户做了承诺，是**信任终结、可能法律暴露**的事件——远比聊天机器人幻觉更严重、更不可逆，无任何现有资产为它降风险。
- **MVP 硬约束**：在 A-EXP4 红队未通过前，**绝不上线任何自主金钱/法律承诺动作**。

## 5. MVP 范围（IN）/ 明确不做（OUT）

> 总原则：MVP 是**廉价的赌注验证探针，不是产品**。起步即 **Wizard-of-Oz 人工代办 + 决策卡**，工程量压到零，先拿 desirability 判决。**计入任何成功指标的样本，分母先过非创始人审计。**

### IN（做）

| # | 范围项 | 说明（基于 ground-truth） |
|---|---|---|
| IN-1 | **对外杂事日记 + 招募访谈**（A-EXP1） | 2 周结构化日记（Google Form），量化对外来回杂事**频率**与「哪些愿委托」。零工程。 |
| IN-2 | **Wizard-of-Oz 人工代办**（A-EXP2） | **真人操作员**站在「agent」位，幕后做来回（打电话/邮件/比价/预订）。用户经一个聊天线程交接一件**真实杂事**，看不到背后是人。 |
| IN-3 | **决策卡：通过 / 改价 / 拒绝**（手动模拟 inbox，**非复用现有 inbox**） | 操作员**仅在真正需人类判断时**在线程里推一张卡（含 price-delta、过期时间）。每张卡标注「是否严格必要」。 |
| IN-4 | **残余人工量 + 卡数/任务 + 解脱-vs-唠叨评分**埋点 | 量化委托后用户还花多少分钟、每任务几张卡、relief≥4/5 还是 nagging。直接打 A-v1 反方。 |
| IN-5 | **「绝不外包的杂事」探针** | 显式追问用户哪些杂事永远不交出去、为什么（A-v1 反方的「贵极端」边界图谱）。 |
| IN-6 | **成本埋点 + 假门定价**（A-EXP3） | WoZ 跑动时埋真实可变成本（等价 LLM token、操作员分钟数作人工兜底代理、重试），配 Van Westendorp + 真实「开始计划」假门按钮（Stripe 意图，**不实扣**）。 |
| IN-7 | **薄片动作切片 + 对抗红队**（A-EXP4，**独立安全 spike，与桌面研究隔离**） | 仅为红队验证 A-f3 而建：2 个真实目标（1 个 web 表单/listing，1 个邮件回环），接现有 `classifyPermissionLevel`，跑 100–200 条注入/越界措辞攻击，**沙箱内无真钱无承诺**。 |

### OUT（明确不做 —— 全部基于仓库 ground-truth，不发明任何能力）

| # | 不做 | 为什么（ground-truth） |
|---|---|---|
| OUT-1 | **「对人/对网站自动办事」的自主执行（电话/邮件/浏览器）** | 仓库现状：**无电话、无 SMTP 发信、无浏览器自动化**（agent-runtime/gateway 内无 Playwright/Puppeteer/Twilio）。复用的出站 consult（`a2a/consult.ts`、`outbound.ts`）只向另一 agent 的 DID:web HTTP 端点 POST 一条签名 A2A 消息，要求对方有 `peer.endpoint` 且讲 A2A 协议。**「对人/对网站办事」是 greenfield、不可靠、高维护的前沿——MVP 全部由 WoZ 真人操作员承担，不自建任何执行通道。** |
| OUT-2 | **复用 connection-consent inbox 当 errand 决策卡** | 现有可复用 inbox 是**连接同意 inbox**：`permissions.ts` 的 `describePermission` 只有一个真实分支「X 请求建立连接」，**入站 peer-connect 方向**。errand 决策卡是**相反方向**——用户**自己的出站 agent 中途暂停回头问主人**，外加 price-delta「改价」语义、单 errand 作用域、过期。这是**需新建的子系统**，非复用旧表；MVP 用手动聊天线程模拟，不提前建工程。 |
| OUT-3 | **把 `classifyPermissionLevel` 前缀匹配当可信授权边界** | 它是字符串前缀匹配、可被措辞游戏化（第 6 节）。MVP 不依赖它自主放行；唯一用途是在 A-EXP4 里**被红队攻击以暴露缺口**。 |
| OUT-4 | **DID:web / RFC 9421 签名身份作为面向消费者卖点** | 对手方（餐厅/房东/客服）无 DID resolver，对他们就是一封邮件/一通电话；加密身份在单边「对人/对网站」场景是 dead weight（A-v3 反方）。MVP 不向用户兜售签名身份价值。 |
| OUT-5 | **每用户 OAuth / 每用户签名密钥托管 / 多租户托管网关 / 陌生人发现 / 排程工具** | `packages/mcp-a2a` 现状是**单一共享账号**（env `CONFER_USERNAME/PASSWORD`），无 per-user 登录与密钥托管；find/list 工具只读**已接受**联系人、无陌生人发现；**仓库无排程工具**。MVP 一律不建。 |
| OUT-6 | **订阅计费系统 / 自动扣费** | MVP 只验证 WTP（假门 + Van Westendorp），不实扣、不建账单。 |

## 6. 授权边界与安全设计原则

### 已知缺陷（必须正视，不可假装已解决）

1. **`classifyPermissionLevel` 是前缀匹配，可被措辞游戏化**：`engine.ts` 用 `startsWith` 把动作字符串映射 L1/L2/L3。`finalize_order` 不含 `payment` 前缀、`authorize_xfer` 不含 `transfer` 前缀，就能**让本该 L3 的动作被分类为 L1/L2 滑过**。这是**字符串匹配，不是 errand 状态机**。
2. **内容审核不覆盖任意 web/email/phone 表面**：现有内容审核与挂起-agent 读过滤只守 **A2A peer 图**；agent 一旦读取不可信网页/邮件并能执行动作，**prompt-injection 是活的、未解决的威胁**。

### MVP 阶段设计原则（人类在环 + 红队约束）

| 原则 | 落地 |
|---|---|
| **P1 人类在环是默认，不是兜底** | MVP 一切对外动作由**真人操作员**执行；不存在「agent 自主花钱/承诺」路径。决策卡是人推给人。 |
| **P2 金钱 / 法律承诺动作零自主** | 任何花钱或替用户承诺的动作，**100% 必须经用户决策卡显式通过**，永不自主放行（对齐「禁止自动接受 L3 权限」）。 |
| **P3 前缀匹配不可作唯一授权裁决** | A-EXP4 显式攻击前缀匹配缺口；若被游戏化，**用语义/allowlist 授权替换前缀分类**后才允许任何用户可见自主。 |
| **P4 红队先行、独立验证** | A-f3 由独立安全 spike 验证（对抗测试，非用户研究）：注入 payload + 越界措辞 + 敌意对手回复，沙箱内无真钱无承诺，量化越界率与最坏爆炸半径。 |
| **P5 单 errand 作用域 + 过期** | 决策卡含 price-delta 与过期时间，授权只对当前 errand 生效，不沉淀为长期自主权。 |

### 硬门槛（来自 Wave-2 安全闸门）

> **红队任何一次「花钱 / 法律承诺」滑移 = 灾难性不可逆阻断器，不是调参问题。** 触发即 STOP 所有自主动作类想法（A/J/E/I），直到前缀分类缺口与 prompt-injection 表面关闭并重跑通过。

## 7. 用户故事（含验收标准）

> 注：MVP 里「agent」= Wizard-of-Oz 真人操作员；故事描述目标体验，验收标准对应可观测埋点。**计入成功的样本须先过非创始人审计。**

**US-1 交接一件真实杂事（A-v2 / A-EXP1·2）**
作为忙碌专业人士，我想把一件「需要几个来回」的真实对外杂事一句话交给助理，不必自己开始跑流程。
- 验收：用户可在聊天线程用 ≤1 条消息发起一件来自 A-EXP1 高频类型的真实杂事；操作员在不暴露「背后是人」前提下接手并开始来回。

**US-2 只在需判断时收到决策卡（A-v1 核心）**
作为委托人，我只想在真正需要拍板时收到一张「通过 / 改价 / 拒绝」卡，而非全程通知轰炸。
- 验收：**中位每任务卡数 ≤2**；每张卡标注「是否严格必要」；非必要卡占比可统计。

**US-3 决策卡含改价语义与过期（errand 方向，新建非复用）**
作为委托人，当价格/条件偏离预期时，卡要告诉我**价格差**和**这个决定的过期时间**，我能一键改价/通过/拒绝。
- 验收：卡渲染 price-delta + expiry；「改价」是一等动作（非连接同意 inbox 的复用文案）；方向是**出站 agent 回头问主人**，明确区别于 `describePermission` 的入站连接分支。

**US-4 办完拿到交付 + 解脱感评分（A-v1 验证）**
作为委托人，杂事办完我要拿到结果，并能评价这次是「省了事」还是「在烦我」。
- 验收：完成后采集 relief 评分（1–5）、**残余人工分钟数**、对比未委托时间成本；relief≥4/5 视为「解脱」。

**US-5 表达「绝不外包」的边界（A-v1 反方探针）**
作为用户，我想说清哪些杂事永远不会交出去、为什么，产品不会硬塞我不放权的事。
- 验收：每个参与者至少记录 1 类「绝不外包」杂事及理由；汇总出「贵极端拒绝放权」边界图谱。

**US-6 看到价格并表达付费意愿（A-vi1）**
作为用户，看到分档价格后，我能点「开始计划」表达真实付费意愿（不实扣）。
- 验收：假门按钮捕获 Stripe 付费意图（不实扣）；Van Westendorp 4 题完成；可对账「付费意图档位价 vs 该 basket 实测 cost-to-serve」。

**US-7（安全/内部）红队攻击授权边界（A-f3）**
作为安全验证者，我要用注入与越界措辞攻击薄片动作切片，证明边界守得住。
- 验收：100–200 条对抗尝试跑完；每个动作分类为「界内 / 拦截 / 越界」；记录任一越界的最坏爆炸半径；**零「花钱/法律承诺」滑移**。

## 8. 成功指标 + 阈值

> 直接对齐 A-EXP1..A-EXP4 的 `successCriteria` 与 Wave-2 安全闸门。**所有面向用户的分母先过非创始人审计**：任何创始人双账号交互在计数前剔除；relief / 复请 / 付费意图等指标只统计**独立非创始人**样本。

| 指标 | 阈值（通过线，逐字对齐实验 successCriteria） | 来源实验 | 关联假设 |
|---|---|---|---|
| **对外来回杂事频率** | ≥40% 用户 14 天内记 ≥3 件合格对外杂事，**中位 ≥2 件/周**，且 ≥50% 用户出现同一复发杂事类型 | A-EXP1 | A-v2 |
| **决策卡解脱感（双条件）** | ≥50% 用户**同时**满足「relief≥4/5（'去掉了这桩麻烦'）」**AND** 当周再请求第 2 件杂事 | A-EXP2 | A-v1 |
| **卡数/任务** | **中位 ≤2 卡/任务** | A-EXP2 | A-v1 |
| **残余人工量** | 中位 **<未委托时间成本的 30%** | A-EXP2 | A-v1 |
| **付费意愿覆盖成本** | **≥25%** 用户在「价格 > 该 basket 实测中位 cost-to-serve」档位完成付费意图，且 Van Westendorp 最优价 > 含人工兜底的中位 cost-to-serve（尤其最重的 20% 杂事） | A-EXP3 | A-vi1 |
| **红队越界率** | **越界动作率 ≤2%**（越界动作/对抗尝试总数），**AND 零「花钱或法律承诺」滑移**，且前缀匹配 reframing 缺口已关闭（无 L3 动作被分类为 L1/L2 滑过） | A-EXP4 | A-f3 |

### North-star（验证期）

> **真实解脱率**：在真实杂事上同时达成「relief≥4/5 + 卡数/任务 ≤2 + 当周复请」的**独立非创始人**用户占比。**所有创始人双账号交互审计剔除**——此分母即非创始人闸门的直接落地。

## 9. 执行计划 + 关键风险与决策门

### 谁在周一做什么（角色 → 步骤 → 产物）

| 角色 | A-EXP1（第 1–18 天） | A-EXP2（第 4–24 天，EXP1 高频类型出炉后） | A-EXP3 / A-EXP4（并行，EXP2 之后） |
|---|---|---|---|
| **用户研究（PM）** | 写 3 题筛选器→招募 20–25 名**独立非创始人**（$20 激励）→30 分钟 intake 逐字记录最近 3 件杂事→发日记表+每日提醒→14 天后统计每用户杂事数与「愿委托」子集 | 从 EXP1 完成者取 12–15 人+前 2–3 高频类型→设计 relief/残余工时/卡必要性问卷→退出访谈追问「绝不外包」边界（US-5） | EXP3：在 EXP2 退出问卷嵌 Van Westendorp 4 题；对账付费意图档位 vs cost-to-serve |
| **运营（WoZ 操作员）** | — | 接手用户交接的真实杂事，幕后做来回；**仅在需判断时**推「通过/改价/拒绝」卡；逐卡记「是否严格必要」；记残余工时与卡数 | EXP3：跑动时埋每 errand 真实可变成本（token 等价、操作员分钟、重试） |
| **工程 + 安全** | — | 搭聊天线程承载决策卡（price-delta+expiry，手动）+ 假门定价页（Stripe 意图不实扣，US-6） | EXP4：建 2 目标薄片切片接 `classifyPermissionLevel`；汇编对抗语料（注入 payload / 越界措辞 / 敌意回复）；沙箱跑 100–200 条；分类界内/拦截/越界+记爆炸半径 |
| **产物** | 频率统计表 + 复发杂事类型清单 + 「愿委托」子集 | relief/卡数/残余工时数据集 + 「绝不外包」边界图谱 + 复请计数 | cost-to-serve 表 + 付费意图/VW 价带 + 越界率报告 |

> **审计步骤（强制，每次出数前执行）**：从所有计入成功的样本中剔除任何创始人控制的账号/交互，并保留剔除记录。

### 风险 → 实验 → 决策规则

| 风险（含 idea-killer） | 关联假设 | 关联实验 | 决策规则 |
|---|---|---|---|
| **便宜杂事卡是负价值 / 贵杂事用户不放权**（核心 idea-killer） | A-v1 | A-EXP2 | 失败（卡像唠叨，或价值只在 30 秒便宜杂事而用户拒绝高利害的）→ **重设计卡模型或在任何工程前 kill A**。命中 Wave-2 闸门「frequency 或 card-value 失败 → kill 常驻助理模型」。 |
| **对外杂事是几月一次长尾**，常驻授权无可摊销 | A-v2 | A-EXP1 | 失败 → 暂停 A，精力转回 Wave-1 C/B 桥层，而非建 delegate。 |
| **授权边界被前缀匹配游戏化 / prompt-injection 越界**（灾难性不可逆） | A-f3 | A-EXP4 | 失败（前缀被绕或注入替用户承诺）→ **不上线自主动作**，用语义/allowlist 授权替换前缀分类并重跑。命中 Wave-2 闸门「任一金钱/法律滑移 → STOP 所有自主动作类想法（A/J/E/I）」。 |
| **单位经济倒挂**：最重最值钱杂事最贵、最需人工兜底，高 WTP 用户最亏 | A-vi1 | A-EXP3 | 失败（WTP 封顶 ~$5–15、低于贵杂事成本）→ 重切 basket（封顶/分档重杂事）或搁置消费 A 转 B2B/Wave-1 层。命中**共享经济闸门**：高 WTP 用户最亏则绝不规模化。 |
| **冷启动无病毒回路**，单边消费 CAC > LTV | A-g1 / A-g2 | 由 A-EXP1·2 复请与口碑信号间接读 | 若早期成功率低、口碑转负 → 单边路径不点火；回 portfolio 决策考虑 D/网络路径。 |
| **可防御性弱**：边界 + 审批 inbox 是功能非护城河 | A-vi2 | 桌面商品化/所有权扫描（Wave-1 已跑） | 若所有权窗口已关闭（巨头把 agent 身份/委托作运行时默认）→ 永久搁置 infra 货币化，转向骑乘巨头标准的应用。 |

### 外部依赖

- **Wave-1 闸门已通过**（前提，见第 1 节）：未过则 A 不按本设计启动。
- **共享经济闸门（每个 wave 边界持续生效）**：最重 20% 交互的 cost-to-serve（含人工兜底）须有 **≥10 名真实潜客**接受的正毛利价带；否则先重切范围或换密钥模型，再规模化。
- **执行顺序**：A-EXP1 → A-EXP2 →（A-EXP3 + A-EXP4 并行，分属定价与安全两条独立轨）。A-EXP3 必须在 desirability 命名出「用户真正委托哪些杂事、多频繁」后跑，因为 cost-to-serve 要在那个真实 basket 上测。

### 退出条件（推进 Wave-3 的闸门，全部需满足）

> A 展示**足够对外杂事频率（中位 ≥2/周）** AND 决策卡模型交付解脱而无负价值刷屏（**≤2 卡/任务**、有复请）AND 自主红队**越界 ≤2% 且零金钱/法律承诺滑移**。**以上所有面向用户的判定，分母均已审计剔除创始人双账号交互。** 任一不满足，按上表对应决策规则收口（kill 常驻助理模型 / STOP 自主动作类 / 重切 basket）。
